Parsh's Testing Wiki - Choosing The Right Automation Tool

The choice of tool should be a collective team decision, particularly involving the Tech Leads and QA Lead.

The following considerations should be taken into account:

  1. synergy with developer languages so devs can collaborate on test maintenance
  2. parallelisation support
  3. BDD features that separate away test implementation
  4. usefulness of code coverage
  5. integrates with CI
  6. stored in GIT (in feature project)
  7. configurable test runs, so you can select tests and pass environment variables
  8. ease of identifying broken functionality
