Top 10 Tips For UI Test Automation

  1. Implement 3 layers of abstraction; the test, the page object, and the browser/device driver. This supports DRY and allows you to switch driver.
  2. Make use of a BDD tool so that tests are readable and focused on Behaviour. The Page Objects should expose a DSL.
  3. Run your application with stubbed APIs so that tests are fast and determinstic. The full regression suite (on one browser) should be fast enough to run as part of CI.
  4. Ensure assertions are expressed in tests and not in framework so that failures are easy to investigate and expectations are visible.
  5. Build a lightweight framework so that adding tests is easy.
  6. Choose a language that Devs are comfortable with so that they can collaborate.
  7. House the tests in the same repo and branch as the feature, so Testability is baked-in.
  8. Use stable and simple locators (css, xpath etc) , ensuring that Devs build applications in support of this.
  9. Avoid flakiness. Before performing your interaction, check elements are; present, visible, in-focus, and clickable with a wait condition. Ask Devs to build a class on the page to indicate if events have completed and have your tests wait on this.
  10. Pick test frameworks that support parallel running and cross browser testing.
