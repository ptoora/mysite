Parsh's Testing Wiki - QA Handover From Dev

Before accepting a Story for exploratory testing, QAs should check the following 4 things as part of a handover from Dev:

  1. a demo that the Story ACs have been met
  2. where the Dev has built the integration tests, we check there is sufficient test coverage
  3. where the QA is building the integration tests , we check that the app is testable and that all AC assertions can be automated (useful IDs and classes in HTML, external dependencies can be injected etc)
  4. scheck that; performance, security, SEO, and logging testability (devops) have been addressed
