Parsh's Testing Wiki - Quality Metrics

The purpose of quality metrics is to provide a trigger for revisiting the SDLC to explore potential weaknesses. Quality is about; defects, timeliness, and cost effectiveness

  • Urgent bugs escaping into production Review => Root cause analysis meetings (PM, Dev, QA)
  • Non urgent bugs escaping into production Review => QA to track and decide best forum
  • Bugs escaping into exploratory testing Review => QA to track and decide best forum
  • CI run time from CI code checkout to completion of automated tests. Review => Aim to fail build if practical and a raise story to fix and relax the limit temporarily
  • Code coverage - unit (code robustness) Review => Fail the build (particularly for new code)
  • Code coverage - functional test (sufficient automation) Review => Fail the build (particularly for new code)
  • Cycle time from story played to story ready for prod Jira
  • Time to recovery Review => fallback Testing
